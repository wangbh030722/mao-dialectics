# Example: Brainstorming Workflow

Prompt:

```text
我想让这个仓库更容易被搜到，帮我头脑风暴一下。
```

Output shape:

## 问题定性

目标不是单纯让项目名被搜到，而是让用户搜索相关问题时能进入仓库。

因此关键词不应只围绕“毛理论”，还要覆盖更大众、更技术化的主题：AI agent skill、Codex skill、Claude Code command、software architecture reasoning、product strategy、prompt engineering、systems thinking、critical thinking。

## 百花齐放

可尝试的入口：

- GitHub Topics
- README 关键词
- docs 页面
- examples 示例
- GitHub Release
- GitHub Pages
- 外部博客
- 社区帖
- awesome list 投稿
- 中英文双语内容

## 去粗取精

优先级最高的是仓库内可控资产：

- `docs/` 搜索落地页
- `examples/` 长尾案例
- README 兼容性和安装说明
- Release 文案
- Topics 建议

## 实践建议

先做仓库内资产，再做外部分发。仓库内容足够清楚后，外部链接才有承接能力。
