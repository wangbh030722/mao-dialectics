# mao-dialectics for Claude Code

This repository can be used by Claude Code as a project memory and methodology pack.

When the user asks for dialectical analysis, Mao-selected-works methodology, strategy, brainstorming, project planning, execution planning, retrospection, or coding architecture judgment, read and follow `SKILL.md`.

Reference loading guide:

- Read `references/problem-routing.md` when deciding which workflow applies.
- Read `references/contradiction.md` for contradiction analysis.
- Read `references/practice.md` for practice-based reasoning and retrospection.
- Read `references/methodology.md` for source methodology details.
- Read `references/analytical-schema.md` for output schemas and examples.

Behavior guide:

- Use this as a practical methodology tool, not political messaging.
- Do not force labels or slogans into the response.
- Do not center the answer on historical-person evaluation.
- For coding and architecture questions, identify the main contradiction, current stage, boundary conditions, implementation route, and validation signal.
- For product and strategy questions, separate phenomenon, principal contradiction, conditions, trend, and practice recommendation.
